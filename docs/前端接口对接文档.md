# 前端接口对接文档

服务地址：`http://127.0.0.1:8080`

## 1. 基础说明

- 除特殊说明外，接口统一返回：

```json
{
  "code": 200,
  "message": "success",
  "rid": "请求ID",
  "data": {}
}
```

- 分页参数统一优先用：
  - `page`
  - `page_size`
- 同时兼容旧参数：
  - `limit`
  - `offset`

- 文档类型：
  - `tender`：招标文件
  - `business_bid`：商务标
  - `technical_bid`：技术标

- 两个特殊接口：
  - `GET /api/postgresql/documents/{identifier_id}/source`：返回 `307` 跳转，不是普通 JSON
  - `GET /api/postgresql/documents/{identifier_id}/preview/pages/{page}`：成功时直接返回预览 JSON，不包 `code/message/data`

## 2. 项目接口

### 2.1 创建项目

`POST /api/postgresql/projects`

请求示例：

```json
{
  "identifier_id": "船体广告"
}
```

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 9,
    "identifier_id": "船体广告",
    "deleted": false
  }
}
```

### 2.2 项目分页列表

`GET /api/postgresql/projects?page=1&page_size=10&keyword=船体`

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 9,
    "page": 1,
    "page_size": 10,
    "items": [
      {
        "id": 9,
        "identifier_id": "船体广告"
      }
    ]
  }
}
```

### 2.3 项目详情

`GET /api/postgresql/projects/{identifier_id}`

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "project": {
      "id": 9,
      "identifier_id": "船体广告"
    },
    "relations": [
      {
        "relation_id": 12
      }
    ]
  }
}
```

### 2.4 修改项目标识

`PUT /api/postgresql/projects/{identifier_id}`

请求示例：

```json
{
  "new_identifier_id": "船体广告_2026"
}
```

### 2.5 删除项目

`DELETE /api/postgresql/projects/{identifier_id}`

### 2.6 批量删除项目

`POST /api/postgresql/projects/batch-delete`

请求示例：

```json
{
  "identifier_ids": ["项目A", "项目B"]
}
```

## 3. 文档接口

### 3.1 上传文档并自动 OCR

`POST /api/postgresql/documents`

表单字段：

- `file`
- `document_type`
- `identifier_id` 可选
- `document_name` 可选

请求示例：

```text
file=@翡翠商务标.pdf
document_type=business_bid
identifier_id=doc-demo-001
document_name=翡翠商务标.pdf
```

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "document": {
      "id": 36,
      "identifier_id": "doc-demo-001",
      "document_type": "business_bid",
      "file_name": "翡翠商务标.pdf",
      "file_url": "minio://update-file/business/doc-demo-001.pdf",
      "extracted": true
    },
    "upload": {
      "bucket_name": "update-file",
      "object_name": "business/doc-demo-001.pdf"
    }
  }
}
```

### 3.2 文档分页列表

`GET /api/postgresql/documents?page=1&page_size=10&document_type=business_bid&extracted=true`

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 36,
    "page": 1,
    "page_size": 10,
    "items": [
      {
        "id": 36,
        "identifier_id": "3029746f-d926-4a8b-9a6a-586ff3396ae7",
        "document_type": "technical_bid",
        "file_name": "翡翠技术标.pdf",
        "extracted": true
      }
    ]
  }
}
```

### 3.3 文档详情

`GET /api/postgresql/documents/{identifier_id}`

说明：

- 会返回完整 OCR `content`
- 列表页不建议直接展示这部分大字段

### 3.4 更新文档

`PUT /api/postgresql/documents/{identifier_id}`

请求示例：

```json
{
  "file_name": "翡翠商务标_修订版.pdf",
  "file_url": "minio://update-file/business/doc-demo-001-v2.pdf"
}
```

### 3.5 删除文档

`DELETE /api/postgresql/documents/{identifier_id}`

### 3.6 批量删除文档

`POST /api/postgresql/documents/batch-delete`

请求示例：

```json
{
  "identifier_ids": ["doc-a", "doc-b"]
}
```

### 3.7 打开原文件

`GET /api/postgresql/documents/{identifier_id}/source?page=12`

说明：

- 返回 `307`
- 前端一般直接 `window.open(url)`

### 3.8 页面预览与高亮

`GET /api/postgresql/documents/{identifier_id}/preview/pages/{page}`

常用查询参数：

- `highlight`
- `highlight_bbox`
- `highlight_rects`

请求示例：

```text
/api/postgresql/documents/41f33eb4-d950-4ec3-ad9d-157426c6ddc7/preview/pages/36?highlight=帐户
```

返回示例：

```json
{
  "page": 36,
  "page_count": 56,
  "image_data_url": "data:image/png;base64,...",
  "highlight_rect_count": 1,
  "highlight_applied": true,
  "document_identifier": "41f33eb4-d950-4ec3-ad9d-157426c6ddc7",
  "file_name": "善元商务标.pdf",
  "source_url": "/api/postgresql/documents/41f33eb4-d950-4ec3-ad9d-157426c6ddc7/source"
}
```

## 4. 项目文件绑定接口

### 4.1 绑定项目文件

`POST /api/postgresql/projects/{identifier_id}/bind-documents`

请求示例：

```json
{
  "tender_document_identifier": "招标文件ID",
  "business_bid_document_identifier": "商务标ID",
  "technical_bid_document_identifier": "技术标ID"
}
```

说明：

- `technical_bid_document_identifier` 现在可以不传
- 适合“先商务、后技术”的流程

### 4.2 绑定关系列表

`GET /api/postgresql/relations?page=1&page_size=10&project_identifier=船体广告`

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 10,
    "items": [
      {
        "relation_id": 12,
        "project_identifier": "船体广告",
        "tender_identifier_id": "03436cf0-46d3-4b42-848a-de9dda29f936",
        "business_bid_identifier_id": "92013c65-0e69-40d0-80f8-656438d745d3",
        "technical_bid_identifier_id": "3029746f-d926-4a8b-9a6a-586ff3396ae7"
      }
    ]
  }
}
```

### 4.3 绑定详情

`GET /api/postgresql/relations/{relation_id}`

### 4.4 更新绑定关系

`PUT /api/postgresql/relations/{relation_id}`

请求体和绑定接口一致。

### 4.5 删除绑定关系

`DELETE /api/postgresql/relations/{relation_id}`

### 4.6 批量删除绑定关系

`POST /api/postgresql/relations/batch-delete`

请求示例：

```json
{
  "relation_ids": [12, 13]
}
```

## 5. 结果接口

### 5.1 结果分页列表

`GET /api/postgresql/results?page=1&page_size=10&keyword=船体`

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 8,
    "items": [
      {
        "id": 47,
        "project_identifier_id": "船体广告",
        "result": {
          "business_bid_format_review": {
            "...": "结果JSON"
          }
        }
      }
    ]
  }
}
```

### 5.2 创建或覆盖项目结果

`POST /api/postgresql/results`

请求示例：

```json
{
  "project_identifier_id": "船体广告",
  "result": {
    "custom_result": {
      "status": "ok"
    }
  }
}
```

### 5.3 查询单个项目结果

`GET /api/postgresql/results/{project_identifier_id}`

### 5.4 更新单个项目结果

`PUT /api/postgresql/results/{project_identifier_id}`

请求示例：

```json
{
  "result": {
    "custom_result": {
      "status": "updated"
    }
  }
}
```

### 5.5 删除单个项目结果

`DELETE /api/postgresql/results/{project_identifier_id}`

### 5.6 批量删除结果

`POST /api/postgresql/results/batch-delete`

请求示例：

```json
{
  "identifier_ids": ["项目A", "项目B"]
}
```

`GET /api/postgresql/projects/{identifier_id}/results`

```json
  "message": "success",
  "data": {
    "project": {
      "identifier_id": "船体广告"
    },
    "result_record": {
      "id": 47,
      "project_identifier_id": "船体广告"
    },
    "results": {
      "bid_document_review": {},
      "business_bid_format_review": {},
      "business_bid_duplicate_check": {},
      "technical_bid_duplicate_check": {}

      "bid_document_review",
      "business_bid_duplicate_check",
      "business_bid_format_review",

  }

```

### 6.2 查询项目单项分析结果

`GET /api/postgresql/projects/{identifier_id}/results/{result_key}`

返回示例：

```json
  "message": "success",
  "data": {
    "project": {
      "identifier_id": "船体广告"
    },
    "project_links": {
```

## 7. 业务分析执行接口

### 7.1 商务标形式审查

`POST /api/postgresql/projects/business-bid-format-review?identifier_id=项目ID`

返回重点：

- `project`
- `result_key`
- `overview`
- `review`
- `result_record`

### 7.2 商务标查重

`POST /api/postgresql/projects/business-bid-duplicate-check?identifier_id=项目ID&max_evidence_sections=5&max_pairs_per_type=0`

返回重点：

- `project`
- `config`
- `groups.business_bid.items`

### 7.3 技术标查重

`POST /api/postgresql/projects/technical-bid-duplicate-check?identifier_id=项目ID&max_evidence_sections=5&max_pairs_per_type=0`

返回重点：

- `project`
- `config`
- `groups.technical_bid.items`

### 7.4 一人多用检查

`POST /api/postgresql/projects/personnel-reuse-check?identifier_id=项目ID`

返回重点：

- `project`
- `config`
- `groups`
- `summary`

### 7.5 错别字检查

`POST /api/postgresql/projects/typo-check?identifier_id=项目ID`

返回重点：

- `project`
- `config`
- `groups`
- `summary`

### 7.6 投标文件综合审查

`POST /api/postgresql/projects/bid-document-review?identifier_id=项目ID&document_scope=all`

返回重点：

- `project`
- `config`
- `groups`

### 7.7 商务标/技术标综合查重

`POST /api/postgresql/projects/duplicate-check?identifier_id=项目ID&document_scope=all&max_evidence_sections=5&max_pairs_per_type=0`

返回重点：

- `project`
- `config`
- `groups.business_bid`
- `groups.technical_bid`

## 8. OCR JSON 直传接口

如果前端拿到的不是原始 PDF，而是 OCR JSON，可以直接调这些接口：

- `POST /api/postgresql/projects/business-bid-format-review/upload-json`
- `POST /api/postgresql/projects/business-bid-duplicate-check/upload-json`
- `POST /api/postgresql/projects/technical-bid-duplicate-check/upload-json`
- `POST /api/postgresql/projects/personnel-reuse-check/upload-json`
- `POST /api/postgresql/projects/typo-check/upload-json`

这几类接口的共同特点：

- 表单上传 JSON 文件
- 会自动创建项目、文档、绑定关系
- 返回结构基本都是：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "project": {
      "identifier_id": "项目ID"
    },
    "result_key": "某个结果键",
    "result": {},
    "result_record": {},
    "document_binding": {}
  }
}
```

## 9. 批量识别接口

### 9.1 批量识别项目文档

`POST /api/postgresql/projects/batch/recognize`

表单字段：

- `tender_file`
- `business_bid_files`
- `technical_bid_files`
- `project_identifier` 可选
- `bid_group_parallelism` 可选

说明：

- 会把招标文件、商务标、技术标一次性全部 OCR
- 如果你想走“先商务、后技术”，不要用这条接口

## 10. 前端推荐对接方式

### 10.1 页面建议

- 项目列表页：`GET /api/postgresql/projects`
- 文档列表页：`GET /api/postgresql/documents`
- 绑定管理页：`GET /api/postgresql/relations`
- 结果页：`GET /api/postgresql/projects/{identifier_id}/visualization-data`
- 原文预览弹窗：`GET /api/postgresql/documents/{identifier_id}/preview/pages/{page}`
- 打开源文件：`GET /api/postgresql/documents/{identifier_id}/source`

### 10.2 节省 OCR 成本的推荐流程

1. 创建项目
2. 上传招标文件
3. 上传商务标
4. 只绑定招标文件和商务标
5. 先做商务标形式审查
6. 再做商务标查重
7. 商务标不过则停止，不上传技术标
8. 商务标通过后再上传技术标
9. 更新绑定关系补上技术标
10. 最后做技术标查重
